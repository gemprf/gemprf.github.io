GEM-pRF

Input Source Selection

Quick Overview

Want to see how GEM-pRF handles different input sources? The demo already has everything set up. Just do this:

  1. Download the GEMpRF DemoKit (it includes sample fMRI/MRI data):
git clone https://github.com/siddmittal/GEMpRF-DemoKit.git
cd GEMpRF-DemoKit
  1. Run the interactive demo:
python run_gemprf_demo.py

The script asks what you want to learn. To explore input sources, pick options 001 - 005. That’s it.

GEM-pRF lets you pull inputs from different places - raw files, BIDS datasets. The goal here is to show you how the software decides where your data is coming from and how to control it.

How Input Sources Work

Everything runs through the XML configuration file. You don’t edit GEM-pRF itself - you just change the config.

  • Filesystem paths: point directly to your fMRI and stimulus files.
  • BIDS datasets: use subject/session selectors, and GEM-pRF finds the files for you.

Learning Through the Demo (001–005)

The DemoKit includes several ready-made examples that show how different input setups look in practice. You don't need to prepare any data - the runner updates paths automatically.

Here’s what the demo script does for you:

  • Interactive menu: lets you select examples 001 - 005, each demonstrating a different input style.
  • Automatic path fixing: the XML files are patched to point at your local copy of the sample data.
  • GPU memory check: if the model won't fit, the script stops early instead of crashing halfway.
  • Full GEM-pRF run: once ready, it calls gp.run() on the chosen configuration.

Examples you will See (001 - 005)

  • 001: BIDS + prfprepare (surface), individual runs
  • 002: BIDS + fMRIPrep (surface), individual runs
  • 003: BIDS + fMRIPrep (volume), individual runs
  • 004: BIDS + fMRIPrep (surface + volume), individual runs
  • 005: Fixed paths (non-BIDS) + prfprepare (surface)

Notes

Make sure GEM-pRF and dependencies (e.g., CuPy for GPU support) are installed. If you're adapting these examples to your own data, the comments in the XML files show exactly which fields control input behaviour.
